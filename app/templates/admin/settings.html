Supermarkt-Einstellungen

Wird in QR-Codes auf Kassenbons gedruckt, damit Smartphone-Kameras beim Scannen direkt den Bon öffnen.
Bei 'Smartphone Web NFC' scannt das alte Smartphone direkt über seinen eingebauten NFC-Chip im Browser!
Wenn aktiviert: Im Leerlauf (kein Bezahlvorgang aktiv) scannt das Terminal-Smartphone NFC-Tags von Produkten und fügt sie automatisch dem Warenkorb auf der Kasse hinzu. Produkte benötigen dafür einen zugewiesenen NFC-Tag (einstellbar bei jedem Produkt). Bei aktivem Bezahlvorgang verhält sich das Terminal weiterhin wie gewohnt.
Bestimmt, ob das Touchscreen-Terminal nach dem Auflegen eine PIN abfragt oder ein Unterschriftenfeld zeigt.
Schaltet das Terminal-Display im Leerlauf automatisch ab/dunkel. Bei Karte auflegen oder Tippen wird es sofort wieder wach.
Bestimmt, ob beim Scannen einer Karte das Banner „NFC Karte erfasst (ID: ...)“ auf dem Terminal auftaucht.

🖨️ Bon-Drucker Hardware-Einstellungen

PDF

⚠️ Wichtig: Docker-Konfiguration erforderlich!

Damit der Docker-Container auf den Drucker zugreifen kann, musst du das Gerät in deiner docker-compose.yml freigeben. Füge dazu den folgenden Block unter web ein:

devices: - "/dev/usb/lp0:/dev/usb/lp0"

Führe anschließend docker compose down && docker compose up -d auf dem Host-System aus.

HINWEIS: Auch wenn der Hardware-Drucker deaktiviert ist, steht der Bon jederzeit als PDF / Ansicht im Browser zur Verfügung!
Schützt vor Papierverschwendung. Einstellbares Stunden-Limit (Standard: 20 Bons/Std).
